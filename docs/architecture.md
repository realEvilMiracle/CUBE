# Photo gallery + Telegram sync (архитектура)

## Цели
- Единое хранилище фото и метаданных для сайта и Telegram-бота.
- Поиск/фильтрация/пагинация по метаданным (дата загрузки, категории, теги, тип файла).
- Экспорт выборок в `zip`.
- Роли пользователей: `admin` и `user`.

## Основные компоненты
- `backend` (Django + DRF): REST API, модель данных, загрузка файлов, поиск/фильтрация, отчеты, генерация `zip`.
- `bot` (Python + aiogram): принимает сообщения Telegram (в т.ч. media groups), пересылает фото и метаданные в backend API, запрашивает выдачи/экспорт.
- `frontend` (vanilla HTML/CSS/JS): загрузка, галерея, фильтры/пагинация, админ UI.

## Поток данных
1. Пользователь сайта грузит фото (`POST /api/photos/upload/`).
2. Backend оптимизирует изображение (lossless-подход: для PNG Pillow-оптимизация, для JPEG не перезапаковываем) и сохраняет файл на SSD (`MEDIA_ROOT`).
3. Backend записывает метаданные в PostgreSQL и сразу делает фото доступным для поиска/галереи.
4. Telegram-бот принимает фото и делает аналогичный `POST /api/bot/upload/` с `X-Bot-Token`.
5. Экспорт: site/bot вызывает `GET /api/photos/export/` или `GET /api/bot/export/`, получает zip.

## Модель доступа
- Сайт: JWT (SimpleJWT).
- Бот: отдельный токен `BOT_API_KEY` в заголовке `X-Bot-Token`.
- `admin`: может управлять категориями/тегами и удалять фото.
